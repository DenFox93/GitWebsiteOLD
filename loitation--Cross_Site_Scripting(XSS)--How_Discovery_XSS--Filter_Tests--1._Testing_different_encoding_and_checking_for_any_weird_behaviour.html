Testing different encoding and checking for any weird behaviour


Finding out what payloads are allowed on the parameter we are testing and how the website reflects/handles it.

• Can I input the most basic HTML Injections like <h2>, <img>, <table> without any filtering and it's reflected as HTML? Are they filtering malicious HTML?
◇ How the website handle encodings such as <%00h2? (There are LOTS to try here, %0d, %0a, %09 etc)
▪ %00 → null character (NUL)
▪ %0D → carriage return
▪ %0A → line feed
▪ %09 → horizontal tab
• If characters are reflected as Character Entities or URL encoded we can test for Double URL Enconding to check how these are handled
example: < is reflected as &lt; or %3C then I will test for double encoding %253C and %26lt;
◇ Characters Entities(w3schools.com)
▪ Less than: < → &lt;
▪ Greater than: > → &gt;
▪ ...
◇ URL encoding (w3schools.com)
▪ Less than: < → %3C
▪ Greater than: > → %3E
▪ Character entity: &lt; → %26lt;
▪ Character entity: &gt; → %26gt;
▪ ...
◇ Double URL encoding
▪ Less than: < → %3C → %253C
▪ Greater than: > → %3E → %253E
• Some interesting encodings to try can be found on https://d3adend.org/xss/ghettoBypass

example:
